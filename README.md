# Applying Forensic Accounting Techniques Using SQL and Python

Demo code and slides for my presentation on forensic accounting techniques. We walk through a fictionalized version of the real Wake County, NC transportation fraud case, applying techniques like Benford's Law, regression analysis, cohort analysis, and gap detection to uncover suspicious patterns in data.

For slides and additional resources, visit https://www.catallaxyservices.com/presentations/forensic-accounting-python/.

## Prerequisites

- [Docker](https://www.docker.com/get-started) and Docker Compose
- (Optional) [uv](https://docs.astral.sh/uv/) for local Python development
- (Optional) [Graphviz](https://graphviz.org/download/) binary for LightGBM tree visualizations when running locally

## Quick Start (Docker)

This is the easiest way to get everything running. Docker Compose starts a SQL Server 2025 instance with the demo database pre-loaded and a Jupyter notebook server with all dependencies installed.

```bash
docker compose up --build
```

Once the services are healthy, open http://localhost:8888 to access the Jupyter notebooks. The notebooks are pre-configured to connect to the SQL Server container.

To stop the containers and clean up when you're done:

```bash
docker compose down
```

To also remove the built images (freeing disk space):

```bash
docker compose down --rmi all
```

If you receive an error removing containers, AppArmor might be blocking the container from stopping.

```bash
sudo aa-remove-unknown
sudo docker copose down
```

## Local Setup

If you prefer to run without Docker, you will need:

1. **SQL Server 2025** (or 2017+). Run the scripts in `code/sql/ForensicAccounting/` in numeric order to create and populate the `ForensicAccounting` database. Note: `00 - Calendar.sql` uses `BULK INSERT` from a local path, so you will need to copy `code/sql/ForensicAccounting/CalendarData.csv` to `/var/opt/mssql/data/CalendarData.csv` (or adjust the path in the script).

2. **Python 3.12+** with [uv](https://docs.astral.sh/uv/):

    ```bash
    cd code/python
    uv sync
    ```

3. **Database connection config**: Create a `config.toml` file in the `code/` directory (the notebooks reference `../config.toml` relative to `code/python/`):

    ```toml
    [database]
    server = "localhost"
    name = "ForensicAccounting"
    username = "sa"
    password = "your_password_here"
    ```

4. **Run Jupyter**:

    ```bash
    cd code/python
    uv run jupyter notebook
    ```

## Notebooks

| # | Notebook | Description |
|---|----------|-------------|
| 0 | Demo 00 - Reviewing a Database | SQL queries for row counts, data types, constraints, and foreign key analysis |
| 1 | Demo 01 - Summary Analysis | Data retrieval, statistics on buses/employees/vendors, cardinality analysis |
| 2 | Demo 02 - Growth Analysis | Active bus trends, invoice/expenditure spikes, Kolmogorov-Smirnov tests, Q-Q plots |
| 3 | Demo 03 - Regression Analysis | Linear regression and LightGBM models to predict expected invoice amounts |
| 4 | Demo 04 - Duplicates and Cardinality | Duplicate invoice detection, clustering around the $999.99 threshold |
| 5 | Demo 05 - Cohorts and Clusters | Box plots, expense category breakdowns, employee clustering, counter-signer analysis |
| 6 | Demo 06 - Benford's Law | Leading digit frequency analysis across multiple data sets |

## Data

The demo data is generated by SQL scripts in `code/sql/ForensicAccounting/`. Because the `Bus` and `LineItem` tables use randomized generation, your data will not be identical to the data shown in the presentation, but the same patterns will be present.
